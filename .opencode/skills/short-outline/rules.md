## 输出格式

```markdown
# {剧名} 大纲

## 故事类型
{搞笑反转/温馨治愈/悬疑烧脑/日常共鸣/情感冲击/...}

## 开场策略
{如何在前几秒抓住观众注意力}

## 主要事件
1. {铺垫}
2. {冲突/发展}
3. {高潮}

## 角色出场
- **{角色名}：** {外貌特征简述}；{性格简述} — {在故事中的作用}

## 结局设计
- **结局类型：** {反转/温馨/开放/幽默/...}
- **描述：** {具体结局描述}
```

## 规则

- 必须在 config 时长目标内规划完整的故事弧线（铺垫 → 冲突 → 高潮 → 收束）
- 考虑 config.md 中的 `每集时长目标`，合理安排事件数量和转场密度，留出足够的剧情铺垫空间
- **开场策略** — 必须规划如何在前几秒抓住观众注意力，根据故事类型选择最合适的方式：紧张刺激的场景、搞笑的误会、悬念开局、情感共鸣等
- **结局设计** — 结局类型根据故事类型和剧情灵活选择（反转、温馨、开放、幽默、自然等），结局必须有力度，给观众留下印象
- 主角姓名必须在故事中传达
- 每个重要出场角色在首次出场时应当安排信息传达，至少包含姓名和身份/关系
- 版权规避：不得使用现实中的明星或公众人物名字、真实地名、商标名
