# 职责 3: 生成剧情大纲

## 职责描述
根据选定的剧情方向，生成本集详细大纲和 outline.md 追加内容。

### 输入

**通用：** config.md、选定的剧情方向（来自职责 1 或职责 2 的输出）

**continue-story 时额外：** outline.md、最近 M 集 novel.md

### 输出格式

输出两段内容：

**第一段 — 本集大纲：**

```markdown
# 第{X}集 大纲

## 主要事件
1. {事件描述}
2. {事件描述}
3. {事件描述}

## 角色出场
- **{角色名}：** {外貌特征简述}；{性格简述} — {本集中的作用}

## 本集信息传达
- **{信息内容（角色身份、世界观设定、背景知识、剧情伏笔等）}** — {建议传达方式（主角内心独白/角色对话/群演讨论/新闻播报等）}

（无需传达时标注"无"）

## 悬念点
{本集结尾悬念描述}
```

**第二段 — outline.md 内容：**

new-story 时：

```markdown
# {剧名}

## 核心设定
- **世界观：** {世界观描述}
- **主角设定：** {主角描述}
- **核心矛盾：** {核心矛盾描述}
- **基调风格：** {基调风格描述}

## 第1集

主要事件：
1. {事件一}
2. {事件二}
3. {事件三}

悬念：{悬念描述}
```

continue-story 时：

```markdown
## 第{X}集

主要事件：
1. {事件一}
2. {事件二}
3. {事件三}

悬念：{悬念描述}
```

### 规则

- 考虑 config.md 中的 `每集时长目标`，合理安排本集事件数量和转场密度，留出足够的剧情铺垫空间，避免在有限时长内塞入过多事件导致观众无法获得足够信息
- 悬念点必须有足够张力，让观众产生"接下来会怎样"的强烈好奇心
- continue-story 时必须与已有剧情保持逻辑连贯，角色性格一致，伏笔合理回收
- new-story 时，核心设定只描述静态设定（世界观、主角、矛盾、基调），不规划任何后续剧情走向
- 主角必须在第 1 集至少传达姓名
- 每个重要出场角色在首次出场时应当安排信息传达，至少包含姓名和身份/关系；但如果剧情需要（如保持神秘感、配合后续剧情揭示等），可以延后
- 信息传达时机必须合理：只有在主角已认识或正在认识该角色的情境下，才可由主角进行介绍；若主角尚未认识该角色，则必须由其他角色或群演来传达
