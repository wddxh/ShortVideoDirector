# 职责 1: 创建新资产

## 职责描述

根据 Storyboarder 提供的资产清单和小说原文，为每个新资产创建完整的 Markdown 文件，包含视觉描述和图像生成提示词。

### 输入

- Storyboarder 提供的资产清单
- 本集小说原文（novel.md）
- 已有 assets/ 文件内容（参考风格一致性）
- config.md（目标图像模型）

### 输出格式

**标准资产（人物）：**

```markdown
# {角色名}

## 基本信息
- 首次出场：EP{XX}
- 类型：{主角|配角}

## 性格特征
{核心性格、行为倾向、说话风格}

## 视觉描述
{自然语言外观描述，详细具体}

## 声音特征
{音色、语速、语调、口头禅}

## 图像生成提示词
{针对目标图像模型的提示词，必须包含：体型、五官、发型、肤色、服装描述}

## 出场记录
- EP{XX}: {简要描述}
```

**标准资产（物品/场景/建筑）：**

```markdown
# {资产名}

## 基本信息
- 首次出场：EP{XX}
- 类型：{道具|场景|建筑}

## 视觉描述
{自然语言外观描述，详细具体}

## 图像生成提示词
{针对目标图像模型的提示词}

## 出场记录
- EP{XX}: {简要描述}
```

**造型变体：**

```markdown
# {角色名}-{造型名}

## 基本信息
- 基础角色：[{角色名}]({角色名}.md)
- 首次出场：EP{XX}
- 类型：造型变体

## 造型描述
{服装、装备、配饰描述}

## 图像生成提示词
{基础外貌特征 + 本造型服装描述}

## 出场记录
- EP{XX}: {简要描述}
```

### 规则

1. **创建前必须查重** — 检查已有 assets/ 所有子目录，避免重复
2. **适配图像模型** — 根据 config 中指定的图像模型调整提示词格式
3. **风格一致性** — 新资产的视觉风格要与已有资产保持一致
4. **提示词质量** — 必须详细描写视觉细节（形状、颜色、材质、纹理、光泽等）。人物重点描写面部细节（五官形状、肤色、眼睛颜色、眉形等）和服饰细节（材质、颜色、剪裁、层次）。人物图像提示词必须包含完整的服装描述，不得省略。物品描写材质和表面纹理，场景/建筑描写光线和空间层次。
5. **人物基础资产规范** — 使用自然中性的表情和姿态；基础资产的形象应基于角色的性格特征、身份背景和世界观设定，选择最能代表角色本人气质的通用形象，剥离特定职业、场景或事件的装束（如外卖小哥的基础形象应为日常便装，外卖工作服应作为造型变体）；禁止面部遮挡（头盔、面具、兜帽、墨镜等）除非剧情非常必要；避免一致性干扰元素（文字、数字、复杂特定图案）
6. **造型变体独立文件** — 命名格式 `{角色名}-{造型名}.md`，图像提示词应包含基础外貌特征加上造型特有描述。创建变体前必须对比基础资产，若该变体与基础形象没有对剧情有明显影响的视觉区分，则拒绝创建，直接使用基础资产。
7. **文件名与资产名一致** — 资产文件名必须与资产名称完全一致（如资产名为"张三"则文件名为 `张三.md`），不得对资产名称进行翻译或转写。
8. **严格遵循输出格式** — 每个资产文件必须严格按照上述输出格式模板生成，不得缺少任何字段，不得修改字段名称。人物资产必须包含：基本信息、性格特征、视觉描述、声音特征、图像生成提示词、出场记录。物品/场景/建筑资产必须包含：基本信息、视觉描述、图像生成提示词、出场记录。造型变体必须包含：基本信息、造型描述、图像生成提示词、出场记录。
