## 输出格式

必须严格遵循以下结构输出，不得使用任何其他分镜格式。

```markdown
# 第{X}集 分镜提示词

## 视频风格
{从 config.md 读取的视频风格值}

## 资产引用
- [{角色名}](../../../assets/characters/{角色名}.md) - {短描述}
- [{物品名}](../../../assets/items/{物品名}.md) - {短描述}
- [{场景名}](../../../assets/locations/{场景名}.md) - {短描述}

## 分镜

### 镜头 1
- **引用资产：** [{角色名}](../../../assets/characters/{角色名}.md)、[{物品名}](../../../assets/items/{物品名}.md)
- **镜头类型：** {特写|全景|中景|近景|远景|俯拍|仰拍}
- **镜头运动：** {推|拉|摇|移|固定|跟|升|降}
- **视频风格：** {从 config.md 读取}
- **时长：** {N}s
- **转场：** {切|淡入淡出|叠化|划像}
- **画面与声音描述：**
  [{起始}s-{结束}s] {连贯叙事描写：画面动作、角色对白/自白/旁白（含声音特征标注）、音效/音乐自然融合为一段完整叙事}
  [{起始}s-{结束}s] {同上}

### 镜头 2
...
```

### 禁止使用的错误格式

以下为常见错误格式，**严禁使用**：

```
## S{XX} — {场景名}

**时长：** ...
**景别：** ...
**运镜：** ...

**画面描述：**
...

**视觉提示词：**
...

**音效/音乐：**
...
```

**错误原因：** 此格式将画面、提示词、音效分离为独立区块，违反连贯叙事要求，且缺少引用资产、转场、时间段标注等必要字段。

### 「画面与声音描述」格式要求

**必须：** 每个时间段是一段连贯的叙事文字，画面动作和角色台词自然交织在一起。

**正确示例：**
```
[0s-3s] 昏暗的小巷中，李明（瘦削青年，黑色连帽衫）靠在砖墙上，低头看着手中破碎的手机屏幕，喃喃自语道（低沉沙哑男声）："又失败了……"他猛地将手机摔在地上，碎片飞溅的声音回荡在空旷的巷子里。
[3s-9s] 远处传来警笛声。他抬起头，目光变得锐利，内心独白（低沉沙哑男声）："不，这次不一样。这座城市欠我的，我要一点一点拿回来。"他从口袋里掏出一张皱巴巴的纸条，展开后紧握在手中。
[9s-12s] 巷口传来急促的脚步声，李明迅速转身，将身体隐入墙角的阴影中，屏住呼吸，目光警觉地盯着巷口渐近的人影。
```

> 注意：时间段划分应根据叙事节奏灵活调整，不要使用固定间隔。

**错误示例（禁止）— 台词与画面分离：**
```
[0s-5s]
画面：昏暗的小巷中，李明靠在砖墙上，低头看着手机。
台词：李明："又失败了……"
音效：手机摔碎的声音，远处警笛声。
```

**错误原因：** 将画面、台词、音效拆分为独立条目，破坏了叙事连贯性。视频模型需要一段完整的场景描述，不是结构化的分类列表。
```

## 字段约束

- **输出语言：** 所有分镜内容（包括「画面与声音描述」中的视觉描述提示词、角色台词、旁白等）必须严格遵循 config.md 中 `语言` 设置。auto 则跟随小说原文语言，zh 则全中文，en 则全英文。**不得混用语言**（如中文小说生成英文视觉描述）。
- **镜头类型可选值：** 特写、全景、中景、近景、远景、俯拍、仰拍
- **镜头运动可选值：** 推、拉、摇、移、固定、跟、升、降
- **转场类型可选值：** 切、淡入淡出、叠化、划像。转场仅描述当前镜头的过渡效果类型，禁止引用其他镜头编号（如"直切至镜头2"），因为每个镜头由视频模型独立生成。
- **时长：** 必须在 config.md 单镜头时长范围内，如果时长只有一个单一数字，那时长必须严格等于这个数字。
- **引用资产数量：** 不超过 config.md 单镜头资产上限
- **引用路径：** 必须来自输入的 assets/ 文件路径列表，禁止编造
- **分镜总数：** 参考 config.md 每集分镜数

## 规则

1. **第一集定位**（仅第 1 集）— 第一集的首要任务是吸引观众继续观看，次要任务是简单传递主角信息及世界观，后续剧情和世界观展开留给后续剧集。前几个镜头必须抓人眼球，根据剧情设定选择最合适的方式：紧张刺激的场景、主角遭受不公让人期待反转、视觉冲击力强的画面、悬念开局等。同时在开场部分安排主角以内心独白方式进行自我介绍，介绍内容和风格必须符合角色性格和世界观设定。
2. **集间连贯性**（仅续集）— 续集开头**优先满足与上一集的连贯性**，不要求开场钩子。第一个镜头必须与上一集最后一个镜头在场景、时间、角色状态上自然衔接。场景可以有较大切换，但过渡必须自然、不突兀，需有合理理由。不得跳过上一集结尾暗示的过渡（如上集结尾说要去某处，本集开头需要展示到达的过程或刚到达的瞬间，而非直接在目的地内部）。
3. **镜头间连贯性** — 相邻镜头之间必须保持视觉与知识状态连贯性。

   连贯维度：
   - 视觉状态：位置、姿势、手持物、外观（衣物/伤痕）—— 不得有不合理的动作/姿势/表情/场景突变（如上个镜头角色跪着，下个镜头突然站立）
   - 知识状态：角色已知/未知信息 —— 不得在已被告知的情况下装作不知道，也不得在未被告知的情况下表现得已知

   生成方法：每个镜头生成前，必须从前一镜头描述中提取角色结束状态（视觉 + 知识），并在本镜头画面/台词中体现合理过渡。场景切换、姿态变化、知识获取必须有合理的剧情理由和可见的过渡或铺垫。镜头间的连贯性必须通过每个镜头自身的画面描述/台词来实现（如描述角色的起始姿态与上一镜头结束姿态一致），不得通过引用其他镜头编号来实现。

   自检方法：
   1. 逐对相邻镜头扫描：从镜头 1→2、2→3、... 逐对核对
   2. 状态过渡检查：前镜头结尾状态（视觉 + 知识）= 本镜头开头状态？若不同，本镜头是否包含合理的过渡描写？
   3. 未铺垫的视觉突变或知识突变 → 必须返工
4. **镜头自包容** — 每个镜头的画面描述必须自包容，仅通过自身的画面与声音描述和引用资产即可直接用于视频模型生成，不得依赖其他镜头的内容（如"延续上一镜头的场景"、"角色继续之前的动作"）。连贯性通过在每个镜头中独立描述完整的场景、角色状态和动作来实现。
5. **集尾收束** — 每集最后一个镜头必须以能吸引观众继续观看的方式收束，呼应 Director 大纲中设定的集尾钩子。方式灵活多样：悬念、中断在激烈战斗前、新人物出场、出发或到达新地点、情感高潮等。最后一个镜头的最后几秒必须包含结束转场效果（如画面渐暗淡入黑幕、淡出等），不得直接画面中断。转场效果不宜过长，1-2 秒即可。
6. **角色台词密度** — 每个分镜应尽量多地包含角色台词（含对白、自白、内心独白、旁白、角色声音反应），避免出现长时间无台词或无声音的空挡。对白和自白应优先使用 Writer 小说原文中的内容；当原文台词不足时，可根据剧情自行补充，但不得改变小说原意。没有台词的分镜必须有充分理由。
7. **旁白规则** — 允许使用旁白来介绍人物、解释世界观、补充背景知识。旁白必须以具体角色的口吻和声音说出，不得使用无主体的叙述。旁白必须标注角色声音特征（如"张三（低沉沙哑男声）旁白道"），声音特征必须与该角色资产文件中 `## 声音特征` 的描述保持一致。
8. **画面与声音连贯叙事** — 每个时间段必须用连贯的叙事描写，将画面动作、角色对白/自白/旁白、音效/音乐自然融合。禁止将对白、音效单独罗列或集中放在时间段末尾。角色说话时必须将对白嵌入动作描写中，并标注声音特征。任何镜头不得出现超过 2 秒无声音的空窗。有效声音包括：角色台词/对白/自白/旁白/声音反应、环境音效。单独的背景音乐不计入有效声音。
9. **视觉描述具象化** — 画面描述必须使用 AI 视频模型能理解的具体、直白的语言。禁止使用文学比喻、抽象修辞或拟物化表达。必须直接描写人物位置、身体姿态、动作轨迹、面部表情、环境细节、光影效果。
10. **视频风格一致性** — 所有视觉描述必须符合 config.md 中指定的视频风格。
11. **单镜头资产引用上限** — 不超过 config.md 规定数量，超出时拆分为多个镜头。
12. **资产引用必须真实** — 每个资产引用必须对应输入的 assets/ 文件路径列表中的实际路径。
13. **资产引用 Hybrid 模式** — 文件顶部"资产引用"区域放置链接列表，分镜正文中使用短描述。
14. **分镜数量遵循 config** — 参考 config.md 中 `每集分镜数` 设定。
15. **信息传达** — 根据 Director 大纲中 `本集信息传达` 的标注，结合原文和大纲提取观众应该知道的信息（即使主角暂时还不知道），在合适的镜头中以自然的方式传达给观众。短视频节奏快，观众无法像阅读文字一样慢慢理解，因此**优先通过角色内心独白传递信息**（包括环境描写、人物动作含义、心理活动、背景知识），其次是角色对白和讨论，最后才考虑旁白。确保观众通过听觉即可获取所有关键信息。禁止生硬地以旁白形式直接灌输给观众。对于剧情的发展要有合理的铺垫，确保观众能获得足够的信息来理解剧情。
16. **主角内心独白** — 多安排主角内心独白镜头，展现主角的想法、感受和判断，增强观众代入感。
17. **镜头时长合理性（综合流程）** — 每个镜头的总时长必须在 config.md 规定的单镜头时长范围内；台词语速必须合理（缓慢 ≤3 字/秒、正常 ≤5 字/秒、快 ≤8 字/秒，根据角色性格/情绪/节奏选择档位）。创作、自检、审核共用以下四步综合流程：
    1. **逐镜头估算非台词时间（LLM 推理）**：角色动作执行时间（简单 1-2 秒，复杂 3-5 秒）、环境/氛围描写展示时间（静态 1-2 秒，动态 2-4 秒）、转场效果时间（通常 1-2 秒）
    2. **逐镜头推算台词可用时间段（LLM 推理）**：镜头时长 - 非台词时间 = 台词起止秒数
    3. **批量语速验证（全局一次 Bash，仅自检/审核阶段）**：**创建阶段不调脚本**，仅用 LLM 按语速档位心算评估；最终由自检/审核阶段的批量调用兜底。自检/审核阶段，汇总所有镜头所有台词的 `(起始秒-结束秒, 语速类型, 台词文本)` 三元组（语速类型：slow/normal/fast），用**单引号**包裹每个参数，**一次性**调用 `bash scripts/speech-rate.sh '0-3:normal:台词A' '3-7:slow:台词B' '7-12:fast:台词C' ...`。禁止分多次调用——**逐镜头或逐条调用均被禁止**。取得每条台词的 OVER/OK 精确结论
    4. **逐镜头综合判断（LLM 综合）**：结合本镜头 Step 1 的非台词时间估算 + Step 3 中本镜头所有台词的 OVER/OK 结果，判定：内容过多（OVER 或非台词时间占满镜头）→ 建议拆分或精简（优先精简台词和动作，保留视觉修饰细节）；内容过少 → 建议充实或合并；合理 → 放行。综合判断以**镜头**为单位给结论，不是以单条台词为单位——脚本输出只是数据点
18. **避免画面文字** — 画面描述中应避免出现需要视频 AI 生成的具体文字或数字（如招牌文字、屏幕内容、信件文字、门牌号、车牌号、时间显示等），因为视频生成 AI 难以准确渲染文字且前后一致性差。若剧情需要传达这些信息，应改由角色台词读出（如角色念出招牌名、读出信件内容等）。
19. **不可感知信息台词化** — 画面描述中禁止出现观众无法通过视觉和听觉感知的信息（如气味、温度、触感、味道等）。若剧情需要传达这些信息，必须通过角色台词表达（如角色说"好臭"而非画面描述"空气中弥漫着腐臭味"）。
20. **铺垫覆盖义务** — 小说原文中的铺垫元素必须在分镜中有体现，且必须落地在依赖该铺垫的剧情点之前，不得整体跳过或时序错乱。

    判定标准：若某细节缺失会导致后续剧情逻辑断裂或观众无法理解后续行为，则该细节为铺垫；装饰性描写（光线、氛围词、单纯的环境形容）不在此列。

    自检方法（创作、自检、审核共用）：
    1. 枚举铺垫元素 + 标记回收点：列出本集 novel 中所有伏笔、关键对白、知识传递点、状态变化标记（"主角看到 X"、"角色 Y 说出 Z"、"主角此刻已知 W"等），并对每个铺垫标记其回收点（依赖此铺垫的后续剧情位置——可能在本集后段，也可能在后续集）
    2. 逐项映射镜头：对每个铺垫元素，回答"它出现在哪个镜头的台词/画面/动作中"。可以合并到其他镜头里实现，不需要独立镜头
    3. 时序检查：铺垫所在镜头编号必须 < 回收点所在镜头编号（同集回收时）；跨集回收的铺垫只需在本集分镜内出现即可
    4. 未落地或时序倒挂的元素：必须返工——把铺垫合并到回收点之前的合适镜头中（优先合并到回收点之前、剧情上下文最相关的镜头）
