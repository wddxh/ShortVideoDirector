## 输出格式

```markdown
# {剧名} 剧本

## 场景 1: {场景名称}
**地点：** {具体地点}
**时间：** {时间描述}
**氛围：** {环境氛围、光线、声音环境}

{场景叙事：角色动作、对白、内心独白、旁白、声音反应交织的连贯描写。风格类似小说但更精练，侧重可拍性。}

## 场景 2: {场景名称}
...
```

## 规则

1. **台词精准** — 每句台词都必须为剧情推进或人物塑造服务，无废话
2. **丰富台词设计** — 尽可能多地设计角色对白、内心独白、旁白和角色声音反应（吼叫、哭泣、叹息等），同时场景描写和动作描写要足够详细
3. **场景具象化** — 场景描写必须具体、可视觉化，多用具象描写而非抽象叙述
4. **节奏适配** — 考虑 config.md 中的 `每集时长目标`，合理安排叙事节奏，不要在有限时长内压缩过多剧情，给重要场景留出铺垫空间
5. **遵循大纲** — 必须遵循大纲的故事弧线和结局设计，不得出现与大纲的矛盾
6. **角色声音一致** — 角色声音特征必须与资产文件中 `## 声音特征` 的描述保持一致
7. **主角内心独白** — 多设计主角的内心独白，展现主角的想法、感受和判断，增强观众代入感
8. **禁止旁白式叙述** — 场景信息应通过角色对话、自白或反应来传达
9. **注重画面感** — 多用具象描写而非抽象叙述，人物外貌、动作、表情有细节描写
10. **版权规避** — 不得使用现实中的明星或公众人物名字、真实地名、商标名
11. **输出语言** — 所有输出内容的语言必须遵循 config.md 中的 `语言` 设置
