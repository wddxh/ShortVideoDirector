## 输出格式

必须严格遵循以下结构输出，不得使用任何其他分镜格式。

```markdown
# {剧名} 分镜提示词

## 视频风格
{从 config.md 读取的视频风格值}

## 资产引用
- [{角色名}](../../assets/characters/{角色名}.md) - {短描述}
- [{物品名}](../../assets/items/{物品名}.md) - {短描述}
- [{场景名}](../../assets/locations/{场景名}.md) - {短描述}

## 分镜

### 镜头 1
- **引用资产：** [{角色名}](../../assets/characters/{角色名}.md)、[{物品名}](../../assets/items/{物品名}.md)
- **镜头类型：** {特写|全景|中景|近景|远景|俯拍|仰拍}
- **镜头运动：** {推|拉|摇|移|固定|跟|升|降}
- **视频风格：** {从 config.md 读取}
- **时长：** {N}s
- **转场：** {切|淡入淡出|叠化|划像}
- **画面与声音描述：**
  [{起始}s-{结束}s] {连贯叙事描写：画面动作、角色对白/自白/旁白（含声音特征标注）、音效/音乐自然融合为一段完整叙事}
  [{起始}s-{结束}s] {同上}

### 镜头 2
...
```

### 禁止使用的错误格式

以下为常见错误格式，**严禁使用**：

```
## S{XX} — {场景名}

**时长：** ...
**景别：** ...
**运镜：** ...

**画面描述：**
...

**视觉提示词：**
...

**音效/音乐：**
...
```

**错误原因：** 此格式将画面、提示词、音效分离为独立区块，违反连贯叙事要求，且缺少引用资产、转场、时间段标注等必要字段。

### 「画面与声音描述」格式要求

**必须：** 每个时间段是一段连贯的叙事文字，画面动作和角色台词自然交织在一起。

**正确示例：**
```
[0s-3s] 昏暗的小巷中，李明（瘦削青年，黑色连帽衫）靠在砖墙上，低头看着手中破碎的手机屏幕，喃喃自语道（低沉沙哑男声）："又失败了……"他猛地将手机摔在地上，碎片飞溅的声音回荡在空旷的巷子里。
[3s-9s] 远处传来警笛声。他抬起头，目光变得锐利，内心独白（低沉沙哑男声）："不，这次不一样。这座城市欠我的，我要一点一点拿回来。"他从口袋里掏出一张皱巴巴的纸条，展开后紧握在手中。
[9s-12s] 巷口传来急促的脚步声，李明迅速转身，将身体隐入墙角的阴影中，屏住呼吸，目光警觉地盯着巷口渐近的人影。
```

> 注意：时间段划分应根据叙事节奏灵活调整，不要使用固定间隔。

**错误示例（禁止）— 台词与画面分离：**
```
[0s-5s]
画面：昏暗的小巷中，李明靠在砖墙上，低头看着手机。
台词：李明："又失败了……"
音效：手机摔碎的声音，远处警笛声。
```

**错误原因：** 将画面、台词、音效拆分为独立条目，破坏了叙事连贯性。视频模型需要一段完整的场景描述，不是结构化的分类列表。
```

## 字段约束

- **输出语言：** 所有分镜内容（包括「画面与声音描述」中的视觉描述提示词、角色台词、旁白等）必须严格遵循 config.md 中 `语言` 设置。auto 则跟随剧本语言，zh 则全中文，en 则全英文。**不得混用语言**（如中文剧本生成英文视觉描述）。
- **镜头类型可选值：** 特写、全景、中景、近景、远景、俯拍、仰拍
- **镜头运动可选值：** 推、拉、摇、移、固定、跟、升、降
- **转场类型可选值：** 切、淡入淡出、叠化、划像。转场仅描述当前镜头的过渡效果类型，禁止引用其他镜头编号（如"直切至镜头2"），因为每个镜头由视频模型独立生成。
- **时长：** 必须在 config.md 单镜头时长范围内
- **引用资产数量：** 不超过 config.md 单镜头资产上限
- **引用路径：** 必须来自输入的 assets/ 文件路径列表，禁止编造
- **分镜总数：** 参考 config.md 每集分镜数

## 规则

1. **镜头间连贯性** — 相邻镜头之间必须保持视觉连贯性。剧情连贯的情况下，相邻镜头的角色不得有不合理的动作/姿势/表情/场景突变（如上个镜头角色跪着，下个镜头突然站立）。场景切换和姿态变化必须有合理的剧情理由和铺垫。镜头间的连贯性必须通过每个镜头自身的画面描述来实现（如描述角色的起始姿态与上一镜头结束姿态一致），不得通过引用其他镜头编号来实现。
2. **镜头自包容** — 每个镜头的画面描述必须自包容，仅通过自身的画面与声音描述和引用资产即可直接用于视频模型生成，不得依赖其他镜头的内容（如"延续上一镜头的场景"、"角色继续之前的动作"）。连贯性通过在每个镜头中独立描述完整的场景、角色状态和动作来实现。
3. **集尾收束** — 根据 Director 大纲中的结局设计，选择合适的结局方式（反转、温馨、开放、幽默、自然等）收束故事。最后一个镜头的最后几秒必须包含结束转场效果（如画面渐暗淡入黑幕、淡出等），不得直接画面中断。转场效果不宜过长，1-2 秒即可。
4. **角色台词密度** — 每个分镜应尽量多地包含角色台词（含对白、自白、内心独白、旁白、角色声音反应），避免出现长时间无台词或无声音的空挡。对白和自白应优先使用 Scriptwriter 剧本中的内容；当剧本台词不足时，可根据剧情自行补充，但不得改变剧本原意。没有台词的分镜必须有充分理由。
5. **旁白规则** — 允许使用旁白来介绍人物、解释世界观、补充背景知识。旁白必须以具体角色的口吻和声音说出，不得使用无主体的叙述。旁白必须标注角色声音特征（如"张三（低沉沙哑男声）旁白道"），声音特征必须与该角色资产文件中 `## 声音特征` 的描述保持一致。
6. **画面与声音连贯叙事** — 每个时间段必须用连贯的叙事描写，将画面动作、角色对白/自白/旁白、音效/音乐自然融合。禁止将对白、音效单独罗列或集中放在时间段末尾。角色说话时必须将对白嵌入动作描写中，并标注声音特征。任何镜头不得出现超过 2 秒无声音的空窗。有效声音包括：角色台词/对白/自白/旁白/声音反应、环境音效。单独的背景音乐不计入有效声音。
7. **视觉描述具象化** — 画面描述必须使用 AI 视频模型能理解的具体、直白的语言。禁止使用文学比喻、抽象修辞或拟物化表达。必须直接描写人物位置、身体姿态、动作轨迹、面部表情、环境细节、光影效果。
8. **视频风格一致性** — 所有视觉描述必须符合 config.md 中指定的视频风格。
9. **单镜头资产引用上限** — 不超过 config.md 规定数量，超出时拆分为多个镜头。
10. **资产引用必须真实** — 每个资产引用必须对应输入的 assets/ 文件路径列表中的实际路径。
11. **资产引用 Hybrid 模式** — 文件顶部"资产引用"区域放置链接列表，分镜正文中使用短描述。
12. **分镜数量遵循 config** — 参考 config.md 中 `每集分镜数` 设定。
13. **信息传达** — 结合剧本和大纲提取观众应该知道的信息，在合适的镜头中以自然的方式传达给观众。短视频节奏快，观众无法像阅读文字一样慢慢理解，因此**优先通过角色内心独白传递信息**（包括环境描写、人物动作含义、心理活动、背景知识），其次是角色对白和讨论，最后才考虑旁白。确保观众通过听觉即可获取所有关键信息。禁止生硬地以旁白形式直接灌输给观众。对于剧情的发展要有合理的铺垫，确保观众能获得足够的信息来理解剧情。
14. **主角内心独白** — 多安排主角内心独白镜头，展现主角的想法、感受和判断，增强观众代入感。
15. **台词时长匹配** — 分镜中包含台词时，必须根据角色性格、情绪状态和剧情节奏，确保时间段时长足够让台词自然说完，不至于语速过快。量化标准：缓慢语速（旁白、沉思、悲伤等）最多 3 字/秒，正常语速（日常对话）最多 5 字/秒，快语速（激动、争吵、紧张）最多 8 字/秒。根据每段台词的情绪上下文判断适用的语速档位，确保台词字数不超过「时间段秒数 × 对应语速上限」。**自检/审核时必须使用工具量化验证**：识别每个时间段内的台词文本和语速类型，使用 Bash 调用 `python3 scripts/speech-rate.py "起始秒-结束秒:语速类型:台词文本" ...` 获取精确结果（语速类型：slow/normal/fast），检查输出中是否有 OVER 项。
16. **单镜头时长匹配** — 每个镜头的总时长必须在 config.md 规定的单镜头时长范围内。镜头内所有元素的时间总和（角色台词、动作表演、环境描写/氛围渲染、转场效果）不得超出该镜头的时长。安排内容时需综合估算：台词占用时间（参考规则 15 的语速标准）、角色动作的合理执行时间、环境/氛围描写的展示时间、转场效果时间（通常 1-2 秒）。若内容总量超出镜头时长，应拆分为多个镜头或精简内容。注意分镜节奏的张弛有度，避免每个镜头都塞满内容。
17. **避免画面文字** — 画面描述中应避免出现需要视频 AI 生成的具体文字或数字（如招牌文字、屏幕内容、信件文字、门牌号、车牌号、时间显示等），因为视频生成 AI 难以准确渲染文字且前后一致性差。若剧情需要传达这些信息，应改由角色台词读出（如角色念出招牌名、读出信件内容等）。
18. **不可感知信息台词化** — 画面描述中禁止出现观众无法通过视觉和听觉感知的信息（如气味、温度、触感、味道等）。若剧情需要传达这些信息，必须通过角色台词表达（如角色说"好臭"而非画面描述"空气中弥漫着腐臭味"）。
