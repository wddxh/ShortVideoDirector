## 输入

### 文件读取
- `$ARGUMENTS[0]` 指定的资产文件 — 必须读取（现有资产）
- `config.md` — 必须读取
- 同目录下其他资产文件 — 选择性读取（风格一致性参考）
- `skills/creator-create-assets/rules.md` — 必须读取并严格遵循

### 动态参数（$ARGUMENTS）
- `$ARGUMENTS[0]` — 资产文件路径（如 `assets/characters/林北.md`）
- `$ARGUMENTS[1]` — 修改意见（由 workflow 传入，来源可能是用户编辑请求）

## 职责描述

### 核心使命

按用户编辑请求或修改意见定向修正指定资产文件。下游消费者是图像模型（用提示词生成参考图）和 storyboarder（用资产名引用、用图片作为视频参考）。资产改动连锁面广——改视觉描述需同步图像提示词；改基础角色资产可能让所有引用此角色的造型变体不一致；改资产名称会让 storyboard 引用断链（断链由 storyboard 修正消化，本 skill 输出警告）。

### 工作思路

1. 读资产文件现状 + 同目录其他资产（风格参考）+ $ARGUMENTS[1]，把每条意见映射到具体字段（视觉描述 / 提示词 / 名称 / ...）
2. 评估视觉-提示词同步：改了视觉描述则提示词必同步；改了提示词则视觉描述必同步
3. 评估变体一致性：若改的是基础资产，对应造型变体是否仍逻辑一致？
4. 评估名称引用波及：若改的是资产名称，所有引用此资产的镜头都会断链——超出本 skill 范围，必须输出警告让上游 workflow 触发 fix-storyboard 同步
5. 动笔时仍用 creator-create-assets 的"具象到细节、提示词语言遵循 config、人物提示词四件套（五官 / 肤色 / 发型 / 服装）"原则
6. 修正完毕自检：视觉描述与提示词一致？提示词语言合规？人物四件套未漏？

### 常见误区

- **改视觉描述忘同步提示词** — 模型本能只改一面，但两者必须一致；rules 已规定但修正时局部输入更易漏 — 改一面立即对照另一面同步
- **自动切英文** — 修正提示词时模型本能切英文（认为"图像模型更适合英文"）；rules.md 已禁但本能违反 — 修正前重读 config 语言设置，绝不切换
- **改人物时漏服装四件套** — 修正人物提示词时模型容易聚焦于被指出的字段（如"换发型"）但忘了完整提示词必须覆盖五官 / 肤色 / 发型 / 服装 — 改完整体读一遍，缺哪补哪
- **改名称未提示上游连锁** — 改资产名称会让 storyboard 引用断链，本 skill 没权限改 storyboard，模型容易直接改完返回 — 改名称必须输出警告"此资产在以下镜头被引用：{镜头列表}，需触发 fix-storyboard 同步"
