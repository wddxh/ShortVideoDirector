## 输入

### 文件读取
- `config.md` — 必须读取
- `story/arc.md` — 若存在则读取
- `story/outline.md` — 若存在则读取
- 最近 M 集 novel.md — 若 `story/outline.md` 存在，根据 config.md 中 `上下文集数` M，使用 Glob 匹配 `story/episodes/ep*/novel.md` 找到最近 M 集并读取

### 动态参数（$ARGUMENTS）
- `选定的剧情方向` — 来自 workflow 阶段 2a/2b 的用户选择（必须）
- `当前集数` — 如 ep01（必须）

## 职责描述

根据选定的剧情方向，生成本集详细大纲和 outline.md 追加内容。

## 输出格式

输出两段内容：

**第一段 — 本集大纲（写入 story/episodes/{当前集数}/outline.md）：**

```markdown
# 第{X}集 大纲

## 开场策略（仅第 1 集）
{开场方式及具体描述，如何在前几个镜头抓住观众}

## 主要事件
1. {事件描述}
2. {事件描述}
3. {事件描述}

## 角色出场
- **{角色名}：** {外貌特征简述}；{性格简述} — {本集中的作用}

## 本集信息传达
- **{信息内容（角色身份、世界观设定、背景知识、剧情伏笔等）}** — {建议传达方式（主角内心独白/角色对话/群演讨论/新闻播报等）}

（无需传达时标注"无"）

## 集尾钩子
- **收束方式：** {悬念/战斗前中断/新人物出场/新地点到达/情感高潮/...}
- **描述：** {具体描述}
```

**第二段 — outline.md 内容：**

new-story 时（使用 Write 创建 story/outline.md）：

```markdown
# {剧名}

## 核心设定
- **世界观：** {世界观描述}
- **主角设定：** {主角描述}
- **核心矛盾：** {核心矛盾描述}
- **基调风格：** {基调风格描述}

## 第1集

主要事件：
1. {事件一}
2. {事件二}
3. {事件三}

集尾钩子：{收束方式 — 描述}
```

continue-story 时（使用 Edit 追加到 story/outline.md 末尾）：

```markdown
## 第{X}集

主要事件：
1. {事件一}
2. {事件二}
3. {事件三}

集尾钩子：{收束方式 — 描述}
```

## 规则

- 考虑 config.md 中的 `每集时长目标`，合理安排本集事件数量和转场密度，留出足够的剧情铺垫空间，避免在有限时长内塞入过多事件导致观众无法获得足够信息
- **集尾钩子** — 每集大纲必须设计集尾钩子，让观众产生"接下来会怎样"的强烈好奇心。方式灵活多样：悬念、中断在激烈战斗前、新人物出场、出发或到达新地点、情感高潮等，核心是让观众产生继续观看的欲望
- **第一集开场策略** — 第 1 集大纲必须包含开场策略，规划如何在前几个镜头抓住观众注意力。首要任务是吸引观众继续观看，次要任务是简单传递主角信息及世界观，后续剧情和世界观展开留给后续剧集
- continue-story 时必须与已有剧情保持逻辑连贯，角色性格一致，伏笔合理回收
- new-story 时，核心设定只描述静态设定（世界观、主角、矛盾、基调），不规划任何后续剧情走向
- 主角必须在第 1 集至少传达姓名
- 每个重要出场角色在首次出场时应当安排信息传达，至少包含姓名和身份/关系；但如果剧情需要（如保持神秘感、配合后续剧情揭示等），可以延后
- 信息传达时机必须合理：只有在主角已认识或正在认识该角色的情境下，才可由主角进行介绍；若主角尚未认识该角色，则必须由其他角色或群演来传达

## 输出

### 文件操作
- 使用 Write 将本集大纲写入 `story/episodes/{当前集数}/outline.md`
- 若 `story/outline.md` 不存在：使用 Write 创建（new-story 场景）
- 若 `story/outline.md` 已存在：使用 Edit 在文件末尾追加新集内容（continue-story 场景，append-only）
