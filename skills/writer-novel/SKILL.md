## 输入文件读取

当前集数由 $ARGUMENTS 传入（如 `ep01`）。

1. 读取 `story/episodes/{当前集数}/outline.md` — 必须读取
2. 读取 `story/outline.md` — 必须读取
3. 读取 `config.md` — 必须读取
4. 若 `assets/characters/` 下有文件（说明非第一集），读取所有角色资产文件（`assets/characters/*.md`）作为角色一致性参考；同时根据 config.md 中的 `上下文集数` M，使用 Glob 匹配 `story/episodes/ep*/novel.md`，找到最近 M 集并读取

## 职责描述

根据本集大纲和整体故事设定，生成具有画面感和紧凑叙事节奏的小说原文。

## 输出格式

```markdown
# 第{X}集

{小说正文，含章节标题}
```

## 输出约束

- 字数范围遵循 config.md 中的 `每集小说字数` 设置（默认 4000-5000）
- 解析规则：范围格式（如 `4000-5000`）直接使用；单个数字（如 `5000`）视为上限，下限自动取 80%
- 场景描写具体可视觉化
- 人物外貌、动作、表情有细节描写

## 规则

1. **呼应集尾钩子** — 每集结尾必须呼应 Director 设定的集尾钩子，留下吸引观众继续观看的结尾
2. **与 outline.md 保持一致** — 不得出现与已有内容的矛盾
3. **人物性格连贯** — 人物性格、说话风格在各集之间保持连贯统一
4. **场景详实** — 场景描写要足够详细，以支撑后续视觉化和分镜制作
5. **用词精炼** — 适合短视频叙事，避免冗长铺陈，同时考虑 config.md 中的 `每集时长目标`，合理安排叙事节奏，不要在有限时长内压缩过多剧情，给重要场景留出铺垫空间
6. **注重画面感** — 多用具象描写而非抽象叙述
7. **丰富台词设计** — 尽可能多地设计角色对白、内心独白和角色声音反应（吼叫、哭泣、叹息等），同时场景描写和动作描写要足够详细
8. **禁止旁白式叙述** — 场景信息应通过角色对话、自白或反应来传达
9. **主角内心独白** — 多设计主角的内心独白，展现主角的想法、感受和判断，增强观众的代入感

## 输出文件操作

使用 Write 将小说原文写入 `story/episodes/{当前集数}/novel.md`。
